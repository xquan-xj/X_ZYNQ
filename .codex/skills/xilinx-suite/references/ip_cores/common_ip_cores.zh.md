# 本工作区已用常见 Vivado IP 核

范围：当前 `_ip` 工程中已经出现的 IP：`clk_wiz` v6.0、`blk_mem_gen` v8.4、`fifo_generator` v13.2、`ila` v6.2、`cordic` v6.0、`xfft` v9.1、`fir_compiler` v7.2。

这份文档用于快速工程判断。最终确认必须看实际 `.xci`、Vivado validation/DRC、综合/实现/时序报告，并在关键参数上回查官方 Product Guide；复杂设计还要仿真和上板观测。

## 当前工程 IP 对照

| IP 核 | 已见模块名 | 用途 |
| --- | --- | --- |
| `clk_wiz` | `clk_wiz_0`, `pll`, `pll_100m` | 基于 MMCM/PLL 的时钟生成、倍频、分频、相位调整 |
| `blk_mem_gen` | `blk_mem_gen_0`, `ram_256x8`, `rom_*` | BRAM RAM/ROM、波表、LCD 字库、坐标轴表 |
| `fifo_generator` | `fifo_generator_0`, `fifo_512x8` | 数据缓冲、流接口缓冲、部分跨时钟解耦 |
| `ila` | `ila_0`, `ila_1` | 片上逻辑分析仪，用于抓真实硬件信号 |
| `cordic` | `cordic_0` | 定点三角/幅相/坐标变换等数学运算 |
| `xfft` | `xfft_0` | AXI4-Stream FFT 频谱处理 |
| `fir_compiler` | `fir_lowpass` | FIR 低通等数字滤波 |

## `clk_wiz` v6.0

常见参数：

| 参数区域 | 意义 | 工程影响 |
| --- | --- | --- |
| 输入时钟频率 | 板上或 PS 输入给 IP 的真实时钟 | 写错会导致生成时钟和时序约束都错 |
| 输出时钟频率 | 希望得到的 PL 时钟 | 必须落在 MMCM/PLL 分频和 VCO 合法范围 |
| 相位/占空比 | 输出时钟边沿位置和高低电平比例 | 视频、ADC、DAC 场景常用，但不能替代时序收敛 |
| 复位极性 | IP reset 端口有效电平 | 必须和 RTL 复位逻辑一致 |
| `locked` | 时钟稳定指示 | 下游逻辑通常应等待它有效后再工作 |

局限和坑：

- 合法频率受 MMCM/PLL VCO、输入频率、分频系数限制。
- Clock Wizard 会生成相关时钟约束，手写 XDC 时不要重复或冲突定义。
- `locked` 只说明时钟稳定，不解决跨时钟域问题。
- 改输出频率会影响下游所有时序、采样率、视频时序、串口参数和 IP OOC 约束。

## `blk_mem_gen` v8.4

常见参数：

| 参数区域 | 意义 | 工程影响 |
| --- | --- | --- |
| 存储类型 | 单口 RAM、简单双口 RAM、真双口 RAM、ROM | 决定端口数量和能否同时读写 |
| 位宽/深度 | 每个地址的数据位数和地址范围 | 决定 BRAM/LUTRAM 资源和地址宽度 |
| 读写模式 | write-first、read-first、no-change | 决定同地址读写时输出什么 |
| 输出寄存器 | 是否寄存输出数据 | 改善时序，但增加读延迟 |
| 初始化文件 | `.coe` 初值文件 | ROM/波表/字库必须保证路径和 radix 正确 |

局限和坑：

- BRAM 读通常是同步读，要预期 1 个或更多周期延迟。
- 同地址读写行为完全取决于模式，仿真和 RTL 预期容易不一致。
- `.coe` 路径在复制工程时很容易失效，应放在 IP 附近并可脚本化重写。
- 小容量存储可能被综合成 LUTRAM，不一定占 BRAM。

## `fifo_generator` v13.2

常见参数：

| 参数区域 | 意义 | 工程影响 |
| --- | --- | --- |
| 公共时钟/独立时钟 | 同步 FIFO 或异步 FIFO | 决定是否可用于数据跨时钟域 |
| 数据位宽/深度 | 每拍数据宽度和可缓存深度 | 决定 BRAM/LUT 资源和抗突发能力 |
| FWFT | First Word Fall Through 首字直通 | 改变 `empty` 和 `dout` 的时序语义 |
| almost full/empty | 提前满/提前空阈值 | 用于提前限流，而不是等硬满硬空 |
| 复位行为 | reset 极性、busy 信号 | 异步 FIFO 尤其要按 IP 要求复位 |

局限和坑：

- 异步 FIFO 适合数据 CDC，不适合随便传控制脉冲。
- `full`/`empty` 不是零延迟反应，控制逻辑要留余量。
- FWFT 模式下读使能和数据有效关系会变，别套普通 FIFO 习惯。
- 双时钟 FIFO 的两个时钟域复位必须满足官方时序要求。

## `ila` v6.2

常见参数：

| 参数区域 | 意义 | 工程影响 |
| --- | --- | --- |
| probe 数量 | 要抓多少组信号 | 增加 probe 会增加布线和调试复杂度 |
| probe 位宽 | 每组 probe 的 bit 数 | 位宽越大越吃 BRAM 和布线 |
| 采样深度 | 保存多少拍数据 | 直接消耗 BRAM |
| 采样时钟 | ILA 用哪个时钟抓信号 | 只能可靠观察同一时钟域信号 |
| 触发条件 | 从什么条件开始抓波形 | 触发条件错会造成误判 |

局限和坑：

- 一个 ILA core 本质上对应一个采样时钟域，跨域信号要同步后再看。
- 大 ILA 会影响布局布线和时序，甚至改变问题表现。
- `mark_debug` 会影响优化，既是工具也是扰动源。
- ILA 看到的是综合实现后的硬件信号，不等同于 RTL 里的所有中间变量。

## `cordic` v6.0

常见参数：

| 参数区域 | 意义 | 工程影响 |
| --- | --- | --- |
| 功能模式 | rotate、translate、sin/cos、atan、sqrt 等 | 决定端口、延迟和数值含义 |
| 数据格式 | 有符号定点、小数/整数位宽 | 决定精度、动态范围和溢出风险 |
| 迭代/精度 | 内部计算迭代深度 | 精度越高通常延迟和资源越高 |
| 流水线 | 是否每级流水 | 提高 Fmax，但增加总延迟 |
| AXI4-Stream 选项 | 流接口握手和附加信号 | 必须和上下游 `valid/ready` 对齐 |

局限和坑：

- 定点缩放必须写清楚；波形像对了，数值比例也可能是错的。
- CORDIC 有固定/可配置延迟，和并行路径必须做延迟对齐。
- 位宽越大资源越高，且可能影响时序。
- 舍入、截断、溢出设置会影响数学精度和频谱质量。

## `xfft` v9.1

常见参数：

| 参数区域 | 意义 | 工程影响 |
| --- | --- | --- |
| FFT 点数 | 一帧参与 FFT 的采样点数 | 决定频率分辨率、延迟、存储和资源 |
| 输入/输出位宽 | 定点数据精度 | 决定噪声底、溢出风险和资源 |
| 缩放策略 | 每级蝶形是否右移 | 防溢出，但会改变输出幅度 |
| 架构 | 流水/突发/资源优化架构 | 在吞吐、延迟、资源之间取舍 |
| 输出顺序 | 自然序或倒位序/倒位组序 | 影响后级频点索引 |
| AXI4-Stream 配置/数据/状态 | 配置通道、数据通道、事件信号 | 握手和 `tlast` 必须严格符合 PG |

局限和坑：

- scaling schedule 是最常见错误来源：能出图不代表幅值正确。
- `tvalid`、`tready`、`tlast`、config channel 时序必须按官方文档。
- FFT 延迟必须在显示、控制、峰值检测路径中显式处理。
- `event_fft_overflow` 等事件信号应在上板调试时接 ILA。

## `fir_compiler` v7.2

常见参数：

| 参数区域 | 意义 | 工程影响 |
| --- | --- | --- |
| 系数 | 滤波器 taps，常来自 `.coe` | 决定频响、阶数和资源 |
| 采样率/时钟率 | 输入样本速率和工作时钟关系 | 决定是否能一拍一个样本 |
| 数据/系数位宽 | 定点精度 | 影响量化噪声、溢出和 DSP 使用 |
| 抽取/插值 | 是否做采样率转换 | 改变输出样本节奏 |
| 舍入/饱和 | 输出数值处理 | 决定截断偏差和削顶行为 |

局限和坑：

- 系数量化后频响会偏离理想滤波器，不能只看浮点设计结果。
- 阶数越高，DSP/BRAM/延迟越高。
- 输出位宽增长要算清楚，盲目截断会损失动态范围。
- 抽取/插值模式会改变 valid 节奏，下游不能按原采样率处理。

## 实用验证清单

- 从 `.xci` 提取实际参数，不靠记忆。
- 生成 output products，检查 Vivado messages。
- 跑综合/实现，检查 black box、critical warning、timing constraint。
- AXI4-Stream IP 必须仿真 `valid/ready/tlast`。
- 数值类 IP 用软件模型比对幅值、相位、频点、滤波输出。
- 调试类 IP 确认 ILA 采样时钟和被观察信号属于同一时钟域。
